Git Config

Learn how to install Git, verify it, and configure your commit identity correctly. This section explains local, global, and system configuration scope so your commits are attributed properly across projects.

Installing Git

Windows, macOS, and Linux

  • Install Git from the official installer for your operating system.
  • After installation, use your terminal to run Git commands locally.
  • Verify installation with:
git --version

If installed correctly, the command prints your Git version.

The git config Command

Configuration levels and behavior

Level Scope When to use
--local Current repository only Project-specific identity or settings
--global Current user on this machine Default identity for all your repositories
--system All users on the machine Machine-wide policy (usually admin-managed)

Set Username and Email

Updated example identity values

Set global identity (recommended baseline):

git config --global user.email "shishir.bansal@example.com"
git config --global user.name "Shishir Bansal"

Verify your configured values:

git config user.email
git config user.name

Local Override Example

Use different identity for one repository

git config --local user.email "work.profile@example.com"
git config --local user.name "Shishir Bansal (Work)"

Local values override global values only inside that repository.

Quick Checklist

Before starting a new repo

  • Git is installed and verified with git --version.
  • Global user.name and user.email are configured.
  • Any project requiring different identity has local overrides.
  • You can read your config values successfully from terminal.
