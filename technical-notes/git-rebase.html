Git Rebase

Rebase creates a clean linear history by replaying commits on top of another branch.

Rebase Commands

Regular and interactive rebase

ActionCommand
Sync branch on maingit rebase main
Pull with rebasegit pull --rebase
Interactive cleanupgit rebase -i HEAD~5
Continue after conflictgit rebase --continue
Abort rebasegit rebase --abort

Conflict + Safety Notes

Avoid risky history rewrites

  • Resolve conflicts file-by-file, then stage and continue.
  • Do not rebase shared protected branches like main.
  • After rebasing pushed commits, update remote with git push --force-with-lease.
